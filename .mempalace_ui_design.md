## Typography System
- Headings: Syncopate (700 weight), uppercase, tight line-height (1.1)
- Body: Inter (300 weight), 1.3rem base, 1.8 line-height
- Monospace accents: Space Mono (700 weight)
- Hero h1: 5.5rem, gradient text (white → 70% white), letter-spacing -2px
- h2: 4rem, letter-spacing -1px
- Kicker: Syncopate 0.85rem, cyan, letter-spacing 5px, with ::before line accent

## Section Architecture (post-overhaul)
1. **Hero** — "LEARNING IS NOW DESIGN" — default card radius
2. **DynamOS** — Full RUA breakdown (once), DynApp pills, radius-dynamos (30px 0 30px 0)
3. **Build** — Creator ecosystem, Nano Banana one-liner, radius-build (0 30px 0 30px)
4. **Tech** — Gem card grid (3-column), radius-tech (0 0 0 30px)
5. **About** — Trimmed 3-paragraph prose + timeline, radius-about (20px 0 20px 0)
6. **Waitlist** — Newsletter signup, radius-waitlist (0 30px 30px 0)

## Key Design Decisions
- **RUA explained ONCE** — Merged Sections 2+3. Nano Banana philosophy moved to Build section as one-liner manifesto
- **Staggered child reveals** — Cards now stagger children (kicker → heading → body → sub-elements) with 100ms delay increments
- **Section-specific card radii** — Each section has distinct border-radius to break monotony
- **Parallax BG** — bg-container gets subtle translateY (0-30px) tied to scroll progress
- **Gem cards** — Tech section uses 3-column grid with accent-color headers, capability tags, hover lift
- **Creator spotlight** — Build section uses 2×2 card grid for DynApp showcase with attribution
- **About trimmed** — 7 paragraphs → 3 tight ones. Timeline carries narrative weight.
- **Footer** — Minimal: brand line + DynApps/Newsletter/Privacy/GitHub links

## RUA Framework (Authoritative Reference)
- **R** — RECOGNIZE: BEEF (Background, Evidence/CHOP, Explanation/SOOI, Feedback)
- **U** — UNDERSTAND: 3C (Collocation, Contexts, Connotation)
- **A** — APPLY: GAP (Gauge, Analyse, Practice)
