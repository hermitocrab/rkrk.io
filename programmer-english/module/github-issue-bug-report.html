Identify the structure and purpose of a well-written GitHub bug report.

github-issue-example.md
1 /* GitHub Issue Example — Bug Report */
2
3 Title: Login button unresponsive on Safari 17 ← Title: specific + platform
4
5 Description ← Description: what, when, where
6 When I click the "Sign In" button on the
7 login page using Safari 17 on macOS, nothing
8 happens. The button appears to be disabled.
9
10 Steps to reproduce: ← Steps: numbered, one action/step
11 1. Open Safari 17 on macOS Monterey 12.6
12 2. Go to app.example.com/login
13 3. Enter valid email and password
14 4. Click the "Sign In" button
15 5. Observe: button click does nothing
16
17 Expected behavior:
18 Clicking "Sign In" should redirect the user ← Expected: what SHOULD happen
19 to the dashboard.
20
21 Actual behavior: ← Actual: what ACTUALLY happened
22 The button is unresponsive. No console errors
23 or network requests are triggered.
24
25 Environment: ← Environment: OS, browser, version
26 - OS: macOS Monterey 12.6
27 - Browser: Safari 17.0
28 - App version: v2.4.1

Exercise 2: Match each section with its purpose.
