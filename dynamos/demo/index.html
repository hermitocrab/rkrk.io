TPACK 形成性评估Formative Assessment

📖 教学法深度 · TPACK × 形成性评估📖 Pedagogy · TPACK × Formative Assessment

📐 理论说什么📐 What the theory says

TPACK：优秀教学 = 技术知识(TK) + 教学法知识(PK) + 内容知识(CK) 的交集。

形成性评估：在教学过程中持续收集学习证据，实时调整教学策略，而非等期末再补救。TPACK: Great teaching = TK + PK + CK intersection.

Formative Assessment: Collect learning evidence during instruction, adjust in real-time — don't wait for finals.

🔧 DynamOS怎么做的🔧 How DynamOS implements it

AI协同仪表盘在老师备课时推荐技术工具（TK），自动匹配教学策略（PK），并关联课程知识点（CK）。课后自动生成形成性评估报告——谁懂了、谁还卡着、下节课该复习什么。Co-teacher dashboard recommends tech tools (TK), matches strategies (PK), links curriculum points (CK). Auto-generates formative reports — who got it, who's stuck, what to review next.

最近发展区ZPD Krashen i+1 艾宾浩斯Ebbinghaus

📖 教学法深度 · ZPD × Krashen × 艾宾浩斯📖 Pedagogy · ZPD × Krashen × Ebbinghaus

📐 理论说什么📐 What the theory says

ZPD：学习发生在"现有水平"与"在帮助下能达到的水平"之间。
Krashen i+1：输入材料的难度应略高于当前水平。
艾宾浩斯：记忆在特定时间点复习效果最优。ZPD: Learning happens between current and potential-with-help levels.
Krashen i+1: Input should be slightly above current level.
Ebbinghaus: Memory optimized by reviewing at specific intervals.

🔧 DynamOS怎么做的🔧 How DynamOS implements it

学生完成能力诊断后，系统定位其在技能树上的ZPD区间，i+1引擎自动调节内容难度。间隔重复系统根据遗忘曲线在最佳时间点推送复习——不是简单的"第二天复习"，而是根据每个学生的遗忘速率个性化计算。After diagnostic, system locates ZPD on skill tree. i+1 engine auto-adjusts difficulty. Spaced repetition pushes review at optimal intervals — personalized to each student's forgetting rate, not a generic "review tomorrow."

🎯 为什么重要🎯 Why it matters

传统课堂一个老师面对30个学生，只能用"平均难度"。DynamOS让每个学生的内容难度都恰好处于i+1。这才是真正的个性化学习——不是把教材换成iPad，而是让内容难度匹配每个大脑。Traditional classrooms serve one average difficulty to 30 students. DynamOS gives every student exactly i+1. This is real personalization — not putting textbooks on an iPad, but matching content difficulty to each brain.

🎯 每个学生24/7拥有私人导师。🎯 Every student gets a private tutor, 24/7.

自主学习SRL 家长参与Epstein

📖 教学法深度 · 自主学习 × Epstein框架📖 Pedagogy · SRL × Epstein Framework

📐 理论说什么📐 What the theory says

SRL (自主学习)：学习者设定目标、选择策略、监控进展、反思调整。
Epstein：家长参与通过六种方式影响学习——养育、沟通、志愿服务、在家学习、决策、社区合作。SRL: Learners set goals, choose strategies, monitor progress, reflect, adjust.
Epstein: Parental involvement works through 6 pathways — parenting, communicating, volunteering, learning at home, decision-making, community collaboration.

🔧 DynamOS怎么做的🔧 How DynamOS implements it

学生端内置目标设定和反思模块（SRL）。课辅端将孩子的学习数据翻译成家长能看懂的语言——不是"正确率82%"，而是"你孩子的弱项是定语从句，本周已练习12次，下周将强化"。同时提供具体可操作的家庭参与建议。Student app has goal-setting and reflection modules (SRL). Parent hub translates learning data into parent-friendly language — not "82% accuracy" but "Your child's weak spot: relative clauses. 12 practice attempts this week. Reinforcement plan next." Plus actionable home involvement suggestions.

掌握学习Bloom 学习分析Siemens

📖 教学法深度 · 掌握学习 × 学习分析📖 Pedagogy · Mastery Learning × Learning Analytics

📐 理论说什么📐 What the theory says

Bloom掌握学习：学生在进入下一个单元前必须掌握当前单元（通常定义80-90%为"掌握"）。
Siemens学习分析：用数据测量、收集、分析学习者行为，以理解和优化学习过程。Bloom's Mastery Learning: Students must master current unit (80-90% threshold) before advancing.
Siemens Learning Analytics: Use data to measure, collect, and analyze learner behavior to understand and optimize learning.

🔧 DynamOS怎么做的🔧 How DynamOS implements it

技能树不是装饰——它是真实的能力图谱。每个节点有掌握阈值。未达标的节点自动锁定后续内容，同时生成针对性补救练习。学习分析引擎追踪40+维度的行为数据，包括答题速度、错误模式、犹豫时间、求助行为等，从数据中发现"假掌握"。Skill tree is a real competency map, not decoration. Each node has a mastery threshold. Unmastered nodes auto-lock next content and generate targeted remedial drills. Analytics engine tracks 40+ behavioral dimensions — answer speed, error patterns, hesitation time, help-seeking — detecting "fake mastery."

🎯 为什么重要🎯 Why it matters

大多数培训机构的"掌握"是：上完课 = 学会了。这导致基础空洞——学生看似一直在学新东西，实际上前面的漏洞越来越大。DynamOS的掌握学习系统杜绝了"假装会了"。Most schools define "mastery" as class-attended = learned. This creates foundation gaps — students look like they're progressing but earlier holes widen. DynamOS mastery system prevents "fake learning."

布鲁姆Bloom TBLT CLT RUA

📖 教学法深度 · 布鲁姆分类学 × TBLT × CLT × 词汇法 × RUA × 对比分析📖 Pedagogy · Bloom's Taxonomy × TBLT × CLT × Lexical Approach × RUA × Contrastive Analysis

📐 理论说什么📐 What the theory says

布鲁姆：认知目标分层——记忆→理解→应用→分析→评价→创造。
TBLT：通过真实任务驱动语言习得。
CLT：语言的核心功能是交际。
词汇法(Lewis)：语言由词汇块(chunks)而非语法规则构成。
RUA：识别→使用→应用的学习循环。
对比分析：利用母语与目标语的异同指导教学。Bloom: cognitive levels — remember → understand → apply → analyze → evaluate → create.
TBLT: real tasks drive acquisition.
CLT: communication is language's core function.
Lexical Approach: language is chunks, not rules.
RUA: Recognize → Use → Apply cycle.
Contrastive Analysis: L1-L2 differences inform teaching.

🔧 DynamOS怎么做的🔧 How DynamOS implements it

内容引擎的每个产出都经过多理论交叉验证：题目是否符合布鲁姆层级？任务设计是否满足TBLT要求（有明确目标、使用目标语言、产出可见成果）？练习是否融入CLT的真实交际场景？词汇教学是否以chunks而非孤立单词为单位？是否遵循RUA循环（先识别句法模式，再用，最后应用）？针对中国学习者的中英对比分析标注。Every content output passes multi-theory validation: Does it match Bloom's level? Does it satisfy TBLT (clear goal, target language, visible output)? Does it embed CLT authentic communicative context? Are vocabulary items taught as chunks not isolated words? Does it follow RUA cycle? Chinese-English contrastive analysis annotations for Chinese learners.

🎯 为什么重要🎯 Why it matters

这是DynamOS与所有"AI教育产品"的最大分水岭。其他产品：把LLM接入，让AI即兴出题——看起来高效，实际上教育学上是一团乱。DynamOS：每个内容输出都必须满足"哪个理论支持这道题的哪个设计"的溯源要求。校长可以放心：你的学生做的每道题，都有教育学依据。This is DynamOS's defining differentiator. Others: plug in an LLM, let AI improvise questions — looks efficient, educationally chaotic. DynamOS: every output must pass "which theory supports which design choice" traceability. School directors can rest easy: every question your students answer has a pedagogical reason.

🎯 每道题都有教育学理由。零AI垃圾。🎯 Every exercise has a pedagogical reason. Zero AI slop.
